## 一、关于资源存放

### 1.1 资源存放的规范

#### 1.1.1 资源存放的目录结构
通过资源加载模块加载的资源，需要将相关的资源放到`Assets/ResourcesAssets`目录下，框架会在这个目录下进行资源的加载和管理。  
并且在这个目录下，建议按照资源类型进行分类存放，比如音频资源放在`Assets/ResourcesAssets/Audio`目录下等。

#### 1.1.2 资源命名的规范
资源不允许出现空格，建议使用下划线`_`或者驼峰命名法来命名资源文件，比如`player_texture.png`或者`PlayerTexture.png`。

### 1.2 资源加载的方式

```csharp
// 加载 Prefab（lifeRef 传 this，资源生命周期绑定当前对象）
// 资源路径为 "父级目录/对象文件名"
// 例如 "Assets/ResourcesAssets/TestSphere"，加载后实例化
// 一般使用这个方式，框架会自动判断使用哪种加载方式（优先使用 YooAsset 加载）
loader.LoadPrefabAsync(this, "TestSphere", (asset, _) =>
{
    var go = Instantiate(asset as GameObject);
});
// 显式指定使用 Resources 加载
loader.LoadPrefabAsync(this, "Config/DefaultConfig", (asset, _) =>
{
    // ...
}, providerType: E_ASSET_PROVIDER.Resources);

// 显式指定使用 YooAsset 加载
loader.LoadAudioClipAsync(this, "Audio/BGM_Main", (asset, _) =>
{
    // ...
}, providerType: E_ASSET_PROVIDER.YooAsset);
```

### 1.4 资源存放总体说明

Art 文件夹：存放美术资源，如图片、模型等。  
ResourceAssets 文件夹：存放通过资源加载模块加载的资源(预制体、音频文件等)，框架会在这个目录下进行资源的加载和管理。  
Resources 文件夹：Unity 内置的资源加载目录，框架不建议直接使用这个，主要用于一些特殊情况的资源存放，比如制作编辑器相关需要使用Res加载。
